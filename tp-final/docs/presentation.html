Oil Production Forecaster

App web para pronóstico de producción por pozo
usando curvas de declinación de Arps
TP Final — IISAIA / FIUBA Ariel Cabello 2026

01Problema y objetivo

Contexto

La producción de un pozo de petróleo decae con el tiempo. Estimar cuánto va a producir a futuro es clave para decisiones económicas: ¿conviene seguir operando?, ¿cuándo abandonar?, ¿cuál es la reserva remanente?

Objetivo

Construir una app web que permita:

  • Cargar historial de producción por pozo desde CSV
  • Ajustar un modelo de declinación de Arps
  • Generar un forecast a N meses y estimar el EUR
  • Visualizar histórico + pronóstico en una sola vista

Entregable

Backend — API REST en FastAPI, SQLite como persistencia, motor de fit en SciPy.

Frontend — SPA en React + TypeScript con vista de upload, listado y detalle de pozo.

Documentación — diagrama de arquitectura del sistema y especificación OpenAPI de la API.

Tests — cobertura mínima 85% en backend (parser + motor de forecast).

CLAUDE.md + rules + skills — instrucciones del proyecto, reglas de estilo y skills reutilizables para asistir el desarrollo con Claude Code.

Fuera de alcance (MVP)

  • Forecast de gas y agua
  • Autenticación / multi-tenant
  • Modelos no-Arps

02Dominio — Curvas de Arps

Modelo empírico estándar de la industria (J. Arps, 1945). Tres variantes según el exponente b:

Exponencial q(t) = qi · exp(−Di·t)   b = 0
Hiperbólica q(t) = qi / (1 + b·Di·t)1/b   0 < b < 1
Armónica q(t) = qi / (1 + Di·t)   b = 1

La armónica decae más lento, la exponencial más rápido; la hiperbólica está entre las dos y suele ser la que mejor ajusta la producción real de pozos no convencionales.

Ajuste
scipy.optimize.curve_fit
con bounds físicos sobre (qi, Di, b)
Métrica de fit
R² sobre el histórico
Salida adicional
EUR — reserva acumulada hasta un cutoff económico

03Stack tecnológico

Backend

Python 3.11+ FastAPI SQLAlchemy 2.x SQLite Pydantic v2 pandas NumPy SciPy pytest ruff

¿Por qué?

  • FastAPI: tipado, OpenAPI auto-generado, async
  • SciPy: curve_fit resuelve el ajuste no-lineal de Arps
  • SQLite: cero infraestructura para un TP académico

Frontend

React 18 TypeScript Vite TanStack Query Recharts Tailwind ESLint Prettier

¿Por qué?

  • Vite + TS: DX rápida, type-safety end-to-end con el backend
  • TanStack Query: cache + fetching declarativo
  • Recharts: gráficos con eje semi-log (clave para curvas de declinación)

Validación
Columnas presentes, fechas ISO, caudales no negativos
Idempotencia
Upsert por well_name; dedupe (well, date)
Feedback
Resumen: pozos nuevos vs existentes, puntos cargados, filas rechazadas

08UI — tres pantallas

Upload

Drag & drop del CSV. Vista previa de las primeras filas, validación inmediata, resumen del resultado de carga.

Wells

Tabla con todos los pozos: último caudal, rango de fechas, número de forecasts. Click → detalle.

Well Detail

Gráfico con histórico + forecast superpuesto. Toggle semi-log. Panel: elegir modelo, horizonte, cutoff → Generar forecast. Tabla de forecasts previos con R² y EUR.
