## 1. Overview

A single-user web app that forecasts per-well oil production with Arps decline curves.

```
┌──────────────┐     HTTP/JSON      ┌──────────────┐    SQLAlchemy    ┌──────────┐
│  Frontend    │  ───────────────▶  │   Backend    │  ──────────────▶ │  SQLite  │
│ React 19 SPA │  ◀───────────────  │   FastAPI    │  ◀────────────── │  (file)  │
│ (port 5173)  │      CORS          │  (port 8000) │                  │          │
└──────────────┘                    └──────────────┘                  └──────────┘
```

No auth, single SQLite file, schema created at startup. CORS restricted to `http://localhost:5173`.

## 2. Backend (`backend/`)

Strict layering — routers do HTTP only, services hold business logic, models are the ORM.

| Layer | Path | Responsibility |
|-------|------|----------------|
| App / lifespan | `app/main.py` | FastAPI app, CORS, router wiring, `init_db()` on startup, `GET /api/health` |
| DB | `app/db.py` | Engine, `SessionLocal`, `get_db` dependency, `init_db` |
| Models (ORM) | `app/models/__init__.py` | `Well`, `ProductionData`, `Forecast`, `ForecastData` |
| Schemas | `app/schemas/__init__.py` | Pydantic v2 request/response models |
| Routers | `app/api/wells.py`, `app/api/forecasts.py` | HTTP endpoints, validation, status codes |
| Services | `app/services/csv_loader.py` | Pure CSV parse + validation (no DB) |
| | `app/services/wells.py` | Upsert/ingest, aggregates, production queries |
| | `app/services/arps.py` | Decline-curve fit / predict / EUR (pure math) |
| | `app/services/forecast.py` | Orchestrates fit + persists forecast series |
| Scripts | `scripts/export_openapi.py` | Dumps the live schema to `docs/openapi.yaml` |

### Request flows

**Upload** (`POST /api/wells/upload`): router reads the file → `csv_loader.parse_production_csv`
(bytes → validated records + rejected rows; missing columns raise → 422) → `wells.ingest_records`
(upsert well by name, upsert production by `(well_id, date)`, last-wins) → `UploadSummary`.

**Forecast** (`POST /api/wells/{id}/forecast`): router validates `model_type` → `forecast.run_forecast`
loads production, converts dates to months-since-first (float), `arps.fit` (bounded `curve_fit`,
hyperbolic falls back to exponential), computes a month limit as last historical point + requested
future horizon, then caps `arps.eur` at the economic cutoff and persists monthly `ForecastData`
points only while the predicted rate is greater than or equal to that cutoff. `ForecastDetail`
responses also filter out any stored legacy points below cutoff → `ForecastDetail`.

## 3. Data model (SQLite)

- `wells` — `id`, `name` (unique), `field`, `operator`, `oil_rate_unit`, `created_at`
- `production_data` — `id`, `well_id` (fk, cascade), `date`, `oil_rate`; unique `(well_id, date)`
- `forecasts` — `id`, `well_id` (fk, cascade), `model_type`, `params_json` (JSON), `r_squared`,
  `horizon_months`, `economic_cutoff`, `eur`, `created_at`
- `forecast_data` — `id`, `forecast_id` (fk, cascade), `date`, `predicted_rate`

All deletes cascade through relationships. No migrations (MVP) — recreate the DB file on schema change.

## 5. Domain — Arps decline curves

Three variants by exponent `b`: exponential (`b=0`), hyperbolic (`0<b<1`), harmonic (`b=1`).
Fit with `scipy.optimize.curve_fit` under physical bounds; time is months-since-first-point (float,
no resampling). EUR integrates the fitted rate (closed-form per variant) to the economic-cutoff time
or the displayed forecast limit, then scales months→days. Rationale and exact formulas:
`.claude/rules/domain-arps.md` and [docs/decisions.md](docs/decisions.md).

## 7. Cross-cutting

- **Config**: backend port 8000; frontend `VITE_API_URL` (`frontend/.env`, default `http://localhost:8000`).
- **CORS**: backend allows `http://localhost:5173` only.
- **Toolchain**: backend via **uv** (Python 3.12 pinned; system `python` is 3.9 — never call it directly);
  frontend via npm. Commands in [CLAUDE.md](CLAUDE.md).
- **Tests**: backend pytest (~95% coverage) over parser, Arps engine, and API; frontend lint + build.

## 8. Risks & constraints (MVP)

- **Hyperbolic convergence**: noisy/short series can defeat `curve_fit`; mitigated by bounds, a
  history-derived initial guess, and exponential fallback.
- **Irregular dates**: time is months-since-first as float; no resampling.
- **EUR ↔ cutoff**: EUR is only comparable across forecasts at equal cutoffs and horizons; the cutoff
  is always stored and also caps the displayed forecast series by predicted rate.
- **Single scale**: only `oil_rate` is forecast; gas/water are out of scope.
- **No migrations / no auth / single user**: deliberate MVP scope.
